☁️ CONFIGURAR SUPABASE

Ingresa las credenciales de tu proyecto Supabase. Las encontrarás en Settings → API en el panel de Supabase.

También necesitas crear las tablas SQL indicadas abajo.

-- Tabla de usuarios
CREATE TABLE IF NOT EXISTS users (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  username text UNIQUE NOT NULL,
  password_hash text NOT NULL,
  created_at timestamptz DEFAULT now()
);

-- Tabla de movimientos
CREATE TABLE IF NOT EXISTS movements (
  id uuid DEFAULT gen_random_uuid() PRIMARY KEY,
  user_id uuid REFERENCES users(id) ON DELETE CASCADE,
  date date NOT NULL,
  type text NOT NULL CHECK (type IN ('income','expense')),
  category text NOT NULL,
  amount numeric(12,2) NOT NULL,
  description text,
  created_at timestamptz DEFAULT now()
);

-- Habilitar RLS
ALTER TABLE movements ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Users own data" ON movements
  USING (user_id = current_setting('app.user_id')::uuid)
  WITH CHECK (user_id = current_setting('app.user_id')::uuid);

-- Función de login
CREATE OR REPLACE FUNCTION authenticate_user(p_username text, p_password text)
RETURNS TABLE(id uuid, username text) AS $$
  SELECT id, username FROM users
  WHERE username = p_username AND password_hash = crypt(p_password, password_hash);
$$ LANGUAGE sql SECURITY DEFINER;

-- Habilitar extensión crypto (para crypt)
CREATE EXTENSION IF NOT EXISTS pgcrypto;

-- Crear primer usuario (reemplaza 'admin' y 'tu_contraseña')
INSERT INTO users (username, password_hash)
VALUES ('admin', crypt('tu_contraseña', gen_salt('bf')));
⬆️ IMPORTAR DATOS JSON

Importa múltiples movimientos desde un archivo JSON. Los datos existentes NO se borran; se añaden los nuevos. Las categorías nuevas que encuentre en el JSON se añaden automáticamente a los selectores.

Formato del JSON:

[
  {
    "date": "2026-03-15",
    "type": "income",
    "category": "Ventas",
    "amount": 1500.00,
    "description": "Ventas del día"
  },
  {
    "date": "2026-03-15",
    "type": "expense",
    "category": "Mi categoría nueva",
    "amount": 300.00,
    "description": "Proveedor frutas"
  }
]

Tipos válidos: income (ingreso) | expense (gasto)
La category puede ser cualquier texto — si es nueva se agrega a la lista.
